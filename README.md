## Описание варианта

**ГОСТ 28147-89** — советский и российский стандарт симметричного блочного шифрования, принятый в 1989 году. Алгоритм работает с блоками по **64 бита** (8 байт) и использует ключ длиной **256 бит** (32 байта).

Основные операции одного раунда:
1. **Сложение по модулю 2³²** — левая половина блока складывается с подключом
2. **S-блок подстановка** — замена каждых 4 бит по таблице (8 S-блоков по 16 элементов)
3. **Циклический сдвиг влево на 11 бит**
4. **XOR** — результат ксорится с правой половиной блока; затем половины меняются местами

Полный алгоритм выполняет **32 раунда**:
- Шифрование: прямые 24 раунда (K₀..K₇ × 3) + обратные 8 раундов (K₇..K₀)
- Дешифрование: прямые 8 раундов (K₀..K₇) + обратные 24 раунда (K₇..K₀ × 3)

ввод текста, ввод ключа, шифрование, дешифрование, копирование результата, очистка полей.

**Криптоядро (UC-09..UC-10)** — внутренние операции класса `GostCipher`:
применение 8 S-блоков подстановки и выполнение 32 раундов преобразования
(24 прямых K₀→K₇ и 8 обратных K₇→K₀).

## Баг-репорт

### BUG-01 (Исправлен)

| Поле | Значение |
|------|----------|
| ID | BUG-01 |
| Обнаружен | TC-09 (автотест) |
| Описание | После расшифровки паддинг не удалялся: возвращался массив с лишними байтами |
| Серьёзность | Critical |
| Причина | Метод `UnpadData` отсутствовал — `Decrypt` возвращал сырой дополненный массив |
| Исправление | Добавлен метод `UnpadData` с PKCS-подобным удалением паддинга |
| Статус |  Исправлен; повторный прогон TC-09 — Passed |

### BUG-02 (Исправлен)

| Поле | Значение |
|------|----------|
| ID | BUG-02 |
| Обнаружен | UI-02 (ручное тестирование) |
| Описание | При пустом поле «Ключ» приложение выбрасывало необработанный `ArgumentException` вместо MessageBox |
| Серьёзность | Major |
| Причина | Отсутствовала проверка `PasswordBox.Password` в `ValidateInput` перед созданием `GostCipher` |
| Исправление | Добавлена проверка `string.IsNullOrWhiteSpace(keyStr)` в метод `ValidateInput` |
| Статус |  Исправлен; UI-01, UI-02 — Passed |
